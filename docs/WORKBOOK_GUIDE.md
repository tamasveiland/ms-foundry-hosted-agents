# AI Agent Monitoring Workbook

This Azure Monitor Workbook provides comprehensive monitoring and analytics for Microsoft Foundry Hosted Agents.

## 📊 Features

### 1. **Overview Dashboard**
- Total conversations
- Success rate
- Average and P95 response times
- Real-time health metrics

### 2. **Recent Conversations**
- Last 20 conversations with details
- Conversation ID, timestamp, duration
- Agent version tracking
- Status indicators (success/failure)

### 3. **Token Usage Analytics**
- Token consumption over time
- Input/output token breakdown
- Usage by model
- Cost estimation support

### 4. **Tool Execution Analytics**
- Tool usage frequency
- Average execution time per tool
- Success rates
- Execution trends over time

### 5. **Performance Breakdown**
- Operation-level latency metrics
- P50, P95, P99 percentiles
- Performance trends

### 6. **Trace Explorer**
- Complete span tree visualization
- Input an Operation ID to see:
  - All LLM calls with token counts
  - Tool executions with parameters
  - Finish reasons and model information
  - Hierarchical parent-child relationships

### 7. **Errors and Failures**
- Failed conversations list
- Exception tracking
- Error patterns and trends

## 📈 Understanding Metrics

### Duration
- All durations are in **milliseconds**
- Color-coded: green (fast) to red (slow)

### Token Usage
- **Input Tokens**: Tokens sent to the model (prompt + context)
- **Output Tokens**: Tokens generated by the model (response)
- **Total Tokens**: Sum of input and output

### Success Rate
- Calculated as: `(Successful Conversations / Total Conversations) × 100`
- Target: > 95%

### Finish Reasons
- **stop**: Natural completion
- **tool_calls**: Model requested tool execution
- **length**: Hit token limit
- **content_filter**: Content policy triggered

## 📚 KQL Query Examples

### Find conversations with specific tool usage
\`\`\`kql
dependencies
| where customDimensions['gen_ai.operation.name'] == "execute_tool"
| where customDimensions['gen_ai.tool.name'] == "multiply"
| summarize count() by bin(timestamp, 1h)
\`\`\`

### Calculate average tokens per conversation
\`\`\`kql
dependencies
| where isnotempty(customDimensions['gen_ai.usage.input_tokens'])
| summarize 
    avgInput = avg(toint(customDimensions['gen_ai.usage.input_tokens'])),
    avgOutput = avg(toint(customDimensions['gen_ai.usage.output_tokens']))
    by bin(timestamp, 1h)
\`\`\`

### Find slow LLM calls
\`\`\`kql
dependencies
| where customDimensions['gen_ai.operation.name'] == "chat"
| where duration > 2000  // More than 2 seconds
| project timestamp, duration, model = tostring(customDimensions['gen_ai.request.model'])
| order by duration desc
\`\`\`

## 🚨 Alerts

Consider setting up alerts for:

| Metric | Threshold | Action |
|--------|-----------|--------|
| Success Rate | < 95% | Investigate failures |
| P95 Duration | > 5000ms | Check performance |
| Error Rate | > 5% | Review exceptions |
| Token Usage | > 100K/hour | Optimize prompts |

To create alerts, use the queries from this workbook in Azure Monitor Alert Rules.

## 🆘 Troubleshooting

### No data showing
- Check time range selection
- Verify agent name parameter matches your agent
- Ensure Application Insights is receiving data

### Missing traces
- Confirm OpenTelemetry instrumentation is enabled
- Check APPLICATIONINSIGHTS_CONNECTION_STRING environment variable
- Verify agent has been invoked recently

### Incorrect metrics
- Refresh the workbook
- Clear browser cache
- Check for data ingestion delays (up to 5 minutes)
